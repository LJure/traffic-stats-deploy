# sing-box 小型管理面板方案

## 目标与边界

在现有 Gin + Vue 流量仪表盘上扩展个人 sing-box 管理能力，通过 Cloudflare Tunnel 对外提供管理站点；sing-box 本身继续仅监听原有代理端口，不为面板新增公网入站端口。

面板服务于单一管理员，不试图复刻 3x-ui。重点是把日常检查、设备管理、Reality SNI 候选验证和受控配置变更收敛到可审计流程。

## 当前已上线的基础

- sing-box 运行 VLESS + Reality，配置文件只允许 root 读取。
- 每台设备有独立认证用户和独立 nftables 双向流量计数器。
- Python 采集器把流量写入 SQLite；Gin/Vue 仪表盘已读取该数据库。
- 仪表盘仅监听 `127.0.0.1:8788`，由 Cloudflare Tunnel 发布。
- sing-box 由 systemd 维护，配置校验命令为 `sing-box check -c /etc/sing-box/config.json`。

## 面板功能

### 只读首页

- sing-box、流量采集器、nftables 规则服务和仪表盘服务的健康状态。
- 代理监听状态、版本、配置摘要（SNI、用户数量、设备名称）。
- 每台设备的流量、最近成功连接时间、最近错误摘要。
- 可过滤的 sing-box 日志；不展示 UUID、Reality 私钥、Tunnel token 或完整订阅链接。

### 受控写操作

- 新增、禁用和删除设备。
- 为指定设备生成节点链接和二维码。
- 预览并切换 Reality SNI。
- 校验配置、安全重启 sing-box、查看并恢复最近备份。

所有写操作必须先展示变更预览，并要求二次确认。配置切换必须采用：备份 -> 生成候选配置 -> `sing-box check` -> 原子替换 -> 重启 -> 健康检查 -> 失败回滚。

### 手动 SNI 扫描

- 仅由管理员在页面点击后触发；没有定时扫描、后台扫描或自动切换。
- 管理候选域名列表，单次选择要检测的候选、并发数和超时。
- 从 VPS 测试 TLS 版本、ALPN、证书 SAN、握手耗时等候选属性，并保存历史结果。
- 扫描只输出排序和建议；选定 SNI 后仍需人工确认受影响设备与新节点链接，才允许切换。
- 扫描任务使用低权限专用进程并限制并发，网页请求不能执行任意 shell 命令或无约束大规模扫描。

## 安全架构

```text
Browser
  -> Cloudflare Access
  -> Cloudflare Tunnel
  -> Gin/Vue (127.0.0.1:8788, trafficstats 用户)
  -> 受限管理任务
  -> root 所有的 singbox-admin 工具
  -> sing-box / nftables / systemd
```

- 管理域名必须启用 Cloudflare Access，只允许管理员身份访问。
- Gin 服务保持非 root；浏览器和 Gin 都不拥有 SSH、sudo 或任意命令执行能力。
- root 管理工具只接受固定操作和经过校验的任务 ID，不接受前端传入的 shell 参数。
- 所有写操作写入审计记录：操作者、时间、变更摘要、校验结果、执行结果与回滚结果。
- 私钥、UUID、Tunnel token、完整导入链接默认不出现在列表、日志或审计详情中。

## 设备与流量模型

新增设备需要同步维护：sing-box VLESS 用户、按认证用户路由的内部 `meter-*` 直连出站、nftables 上传/下载计数器以及采集器身份映射。后续提供 `add-device` 管理操作或脚本，一次完成上述四层变更并生成节点链接，避免手工遗漏。

Mux 是客户端节点设置，不是当前服务端可统一切换的参数。面板可以记录建议和在生成节点时写入兼容参数，但不应把它实现为服务端“一键开关”。

## 实施顺序

1. 只读管理页：服务状态、配置摘要、最近连接、错误日志和流量复用现有仪表盘。
2. 受控执行器：配置备份、校验、重启、健康检查、回滚和审计。
3. 设备生命周期：新增、禁用、链接/二维码生成。
4. 手动 SNI 扫描与候选历史。

每一阶段单独发布和验证；只读页面先上线，写操作不与其混合发布。
